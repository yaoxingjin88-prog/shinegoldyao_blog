# 个人技术博客与作品集门户 - 设计文档 v2.0

## 一、项目概述

### 1.1 项目定位
高级个人技术博客与作品集门户，作为前端架构师、在校大学生及兼职软件开发者的核心名片，用于：
- 沉淀技术文章
- 展示软件开发实力
- 展示开源项目
- 记录职业轨迹

### 1.2 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                        用户层                                │
├─────────────────────┬───────────────────────────────────────┤
│   前台博客 (C端)     │         后台管理 (B端)                 │
│   Nuxt 3 SSR/SSG    │         Vue 3 SPA                     │
│   Tailwind CSS      │         Element Plus                  │
│   shadcn-nuxt       │         Vue Router                    │
├─────────────────────┴───────────────────────────────────────┤
│                      API 网关层                              │
│                   NestJS + Swagger                          │
├─────────────────────────────────────────────────────────────┤
│                      数据持久层                              │
│              Prisma ORM + MySQL 8.0                         │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 技术栈详情

| 端 | 技术栈 |
|---|---|
| 前台博客 | Nuxt 3 + TypeScript + Tailwind CSS + shadcn-nuxt + Pinia + Axios + @nuxt/image + @nuxtjs/seo + Giscus |
| 后台管理 | Vue 3 (Vite) + TypeScript + Element Plus + Vue Router + Pinia + Axios + md-editor-v3 |
| 后端服务 | NestJS + TypeScript + MySQL 8.0 + Prisma ORM + JWT + Swagger + @nestjs/throttler |

### 2.2 完整数据表清单

| 表名 | 说明 | 核心字段 |
|---|---|---|
| site_config | 网站全局配置 | config_key(unique), config_value(longtext) |
| banner | 首页轮播图 | title, image_url, link_url, sort, is_enable |
| skill_category | 技术栈分类 | category_name, theme_color, sort |
| skill | 技术栈详情 | skill_name, proficiency(0-100), icon_url, category_id |
| article_category | 文章分类 | category_name, slug(unique+delete_time) |
| article_tag | 文章标签 | tag_name, slug, theme_color |
| article | 文章主表 | title, slug, content(md), html_content, view_count, is_top, is_publish |
| article_tag_relation | 文章标签关联 | article_id + tag_id (复合主键) |
| project | 项目作品集 | project_name, short_desc, tech_stack(JSON数组), demo_url, github_url, gitee_url |
| social_link | 社交链接 | platform_name, icon_url, link_url |
| message | 用户留言 | nickname, email, content, ip_address, user_agent, is_read |
| experience | 经历记录 | type(1=教育/2=工作), org_name, position, start_date, end_date |
| admin_user | 管理员 | username, password(bcrypt), nickname, avatar, last_login_time |

### 2.3 软删除策略
所有业务表统一使用 `delete_time` (bigint UNSIGNED) 实现软删除：
- `0` = 未删除（查询条件 `WHERE delete_time = 0`）
- `>0` = 删除时间戳（毫秒级 Unix 时间戳）

### 2.4 site_config 初始键值

| config_key | 说明 |
|---|---|
| site_title | 网站标题 |
| site_subtitle | 网站副标题 |
| home_intro | 首页个人简介 |
| about_content | 关于我页面富文本 |
| avatar | 个人头像地址 |
| seo_keywords | 全局SEO关键词 |
| seo_description | 全局SEO描述 |

## 三、UI 设计规范

### 3.1 主题系统

#### 亮色主题（Light）
```css
body { background: #f9fafb; color: #111827; }
--card: #ffffff
--border: #e5e7eb
--primary: #06B6D4  /* cyan-500 */
--accent: #8B5CF6   /* purple-500 */

/* 背景光晕：bg-purple-300/40 + bg-cyan-300/40 */
```

#### 暗色主题（Dark）
```css
body { background: #050505; color: #e5e7eb; }
--card: rgba(255,255,255,0.02)
--border: rgba(255,255,255,0.1)
--primary: #22D3EE  /* cyan-400 */
--accent: #A78BFA   /* purple-400 */
/* 导航栏滚动后：bg-[#050505]/80 backdrop-blur-lg border-b border-white/10 */
/* 背景光晕：bg-purple-600/20 + bg-cyan-600/10 */
```

### 3.2 动效规范
- **页面进入**：`fadeInUp` (translateY 20px → 0, opacity 0 → 1, 0.8s)，各区块 0.1s 间隔 delay
- **卡片悬停**：`hover:-translate-y-1 hover:shadow-xl transition-all duration-500`
- **导航栏**：滚动超过 50px 切换为毛玻璃效果
- **轮播图**：5s 自动播放，悬停暂停，1000ms transition

### 4.2 统一响应格式

```typescript
// 成功响应
interface ApiResponse<T> {
  code: 0;
  message: 'ok';
  data: T;
}

// 分页响应
interface PageData<T> {
  list: T[];
  total: number;
  page: number;
  pageSize: number;
}

// 错误响应
interface ErrorResponse {
  code: number;   // 40001=参数错误, 40100=未登录, 40300=无权限, 50000=服务器错误
  message: string;
  data: null;
}
```

### 4.3 文章列表查询参数
```
GET /api/article?page=1&pageSize=10&categoryId=1&tagId=2&keyword=vue&isTop=1
```

## 八、图片存储策略

**开发阶段**：使用 multer 存储到本地 `server/uploads/` 目录，通过 NestJS 静态资源服务暴露。

**生产阶段（推荐）**：迁移至对象存储，可选：
- 阿里云 OSS
- Cloudflare R2（免费额度大）
- MinIO（自建）

图片 URL 统一通过后端接口返回，前端无需关心存储实现。

## 九、搜索实现方案

文章列表支持关键词搜索：
- **开发阶段**：MySQL `LIKE '%keyword%'`（简单，无需额外配置）
- **生产阶段（可选升级）**：集成 Meilisearch，文章发布时同步索引

当前 API 使用 `keyword` 参数对 `title`、`summary` 字段进行 LIKE 模糊搜索。

## 十、安全策略

1. **认证**: JWT accessToken(7d) + refreshToken(30d) 双令牌机制
2. **密码**: bcryptjs 10轮加密存储
3. **接口**: 全局 ValidationPipe（whitelist + forbidNonWhitelisted）
4. **防刷**: `@nestjs/throttler` 留言接口限制 5次/60s（按IP）
5. **CORS**: 白名单域名配置（生产环境严格限制）
6. **XSS**: Markdown 解析后的 HTML 通过 sanitize-html 过滤
7. **文件上传**: 限制类型（image/\*）+ 大小（5MB）

## 十一、部署架构

```
┌─────────────────────────────────────────────────────────────┐

│            SSL证书: Let's Encrypt (Certbot自动续签)           │

│  yourdomain.com     │       admin.yourdomain.com            │

├─────────────────────┴───────────────────────────────────────┤
│                 api.yourdomain.com                          │
│            (NestJS API, PM2 进程守护)                        │

│              MySQL 8.0（本机 or Docker）                     │

CI/CD (GitHub Actions):
  main分支push → 运行测试 → SSH部署到服务器 → PM2 reload
```

### Nginx 配置要点
```nginx
# 前台 SSR（Node进程）
location / {
  proxy_pass http://localhost:3001;
}
# 后端API
location /api {
  proxy_pass http://localhost:3000;
}
# 上传文件静态服务
location /uploads {
  alias /var/www/tech-blog/server/uploads;
}
```
